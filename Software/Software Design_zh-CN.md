## 架构概览

PixPill 固件采用**裸机循环**，不使用 RTOS。主循环由 HAL 延时驱动，按固定帧率运行粒子模拟、手势检测。

```
main.c → cpp_main()

Components/
├── Src/
│   ├── cpp_main.cpp     ← 状态机、手势检测、主循环 (约 350 行)
│   ├── bma530.cpp       ← BMA530 I²C 加速度计驱动
│   ├── is31fl3736.cpp   ← IS31FL3736 I²C LED 矩阵驱动 (分页寄存器)
│   ├── sandsim.cpp      ← 沙粒模拟 (元胞自动机，32 粒子)
│   ├── liquidsim.cpp    ← 液体模拟 (类 SPH，16 粒子)(粒子较少是为了减少计算量，再多会不够流畅)
│   └── pixpill_anim.cpp ← 开机 / 关机 / 充电 / 故障动画
└── Inc/                 ← 头文件 + 静态查找表 (constexpr LUT)
```

### 系统资源

| 资源 | 占用 | 说明 |
|------|------|------|
| **RAM** | 2872 B / 6 KB (46.74%) | 含栈、粒子缓冲区、查找表 |
| **FLASH** | 32656 B / 32 KB (99.66%) | -Os 优化后极为紧凑，FLASH使用率将近100% |
| **I²C** | I2C1 (PB6-SCL, PC14-SDA) | 挂载 BMA530(地址0x18 << 1) + IS31FL3736(地址0x50 << 1) |
| **TIM3_CH2** | PA7 — LED_STATUS 呼吸灯 | 1kHz PWM, sin平方 64 步查表 |
| **GPIO** | PA8 (ERR), PB7 (CHG), PC15 (SHPACT) | nPM1100 监控与控制 |

FLASH 约有三分之一的空间被LUT占用。在 C011 48MHz 计算速度的限制下，使用了较多查表以提高液体的模拟流畅性。

## 状态机

```
 [上电 / 船运唤醒]
        ↓
      BOOT -> 开机动画（"PIXPILL" 滚动字幕）
        ↓
      RUNNING -> 物理模拟（默认：液体模式）
        | 570ms内摇晃4次  -> 切换沙/液
        | 静止 22s       -> SHUTDOWN
        | nPM ERR       -> 故障处理
        | USB 插入       -> 充电指示 4s 后返回
        ↓
      SHUTDOWN -> 关机动画 -> SHPACT 拉高 -> 进入船运模式
```

- **状态切换仅在主循环入口处判断**，避免中途意外跳转
- **电源 ERR 状态**优先级最高——触发时暂停模拟，LED 阵列闪烁报警，等待 ERR 清除后恢复

## 沙粒模拟（class SandSim）

基于经典 **falling-sand 元胞自动机**算法。

### 核心参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 粒子数 | 32 | 在 96 个 LED 格点上移动，带有速度和加速度 |
| 重力方向 | 8（上下左右 + 四角） | 基于 BMA530 加速度计重力方向进行八个重力扇区的判定 |
| 扫描顺序 | 4 种（下/右/右下/右上） | 配合重力方向逐行扫描，避免重复处理 |
| 速度等级 | 0-5 级 | 粒子可积累动量加速下落 |

### 实现

- **邻居表**：96×8 静态查找表，存储每个格点 8 个方向的邻居索引，O(1) 访问
- **扫描表**：根据重力方向选择扫描顺序，保证粒子正确堆叠不穿透
- **速度机制**：粒子在下落方向积累速度（最多 5 步），倾斜角度越大下落越快
- **帧率自适应**：倾斜角度越大帧率越高，静置则减速

## 液体模拟（class LiquidSim）

类 SPH 的实现，计算密度场压力和粒子间作用力。

### 核心参数

| 参数 | 最终值 | 说明 |
|------|--------|------|
| 粒子数 | 16 | 每个粒子对应多个 LED 亮度 |
| SUBSTEPS | 2 | 每帧多次子步积分可提高稳定性 |
| GRAVITY | 1.4 | 重力加速度 |
| MIN_DIST | 1.24 | 粒子最小距离（碰撞阈值） |
| DAMPING | 0.88 | 速度阻尼系数 |
| COLLISION_DAMPING | 0.5 | 碰撞能量损失 |
| WALL_STRENGTH | 1.4 | 边界反弹力 |
| THRESHOLD | 95 | 密度压力触发阈值 |
| FORCE | 3.6 | 密度压力系数 |

### 实现要点

- **密度场**：18×6 网格，每个粒子的密度贡献用**双线性插值**分配到上下左右 4 个最近格子
- **压力**：密度超过阈值时产生向外推力，推动粒子分布更均匀
- **碰撞检测**：粒子间距离 < MIN_DIST 时施加斥力
- **边界处理**：粒子碰到 LED 阵列边界时反弹，带阻尼
- **坐标处理**：BMA530 加速度计与 LED 阵列背对背安装，gravity.x/y 需取反

### 调参流程

桌面调参脚本 `liquid_visual_sim.py` 可先在 PC 上可视化模拟效果，调好参数再同步到 C++ 固件。参数最终值经过多轮迭代确定。

## 手势识别

通过 BMA530 加速度计 **X 轴（上下）方向变化计数**实现摇晃手势检测。

### 逻辑

```
阈值 SHAKE_THRESHOLD = 14000（约 ±0.9g）
窗口 SHAKE_WINDOW_MS = 570 ms
目标次数 = 8 次方向变化（即上下摇晃 4 个来回）

1. 每一帧读取加速度 X 轴原始值
2. 超过 ±14000 视为有效方向信号
3. 方向翻转时计数 +1
4. 若 570ms 内完成 8 次翻转 → 触发模式切换
5. 超时则重置计数
```

### 设计考量

- **窗口时间较短（570ms）**：避免正常把玩时误触发
- **阈值较高（14000）**：需要明显、快速的摇晃动作
- **仅使用 X 轴**：减少误触发概率，上下摇晃最自然

## 电源管理

### nPM1100 船运模式

- **进入条件**：静止 22 秒
- **静止检测**：逐帧比较 BMA530 加速度变化量。新增 `MOTION_DELTA_THRESHOLD=200`，任何姿态下加速度差值低于阈值即视为静止
- **关机流程**：播放关机动画 -> SHPACT 拉高（此时若未充电，3.0V VOUTB将被关闭，MCU将断电） -> STOP 模式

### 充电检测

- USB 插入时 `nPM1100 CHG` 引脚拉低
- **充电动画**：电池图案 + LED_STATUS 呼吸灯，播放 4 秒后恢复正常模拟
- **拔电后**：充电动画结束，继续正常运行

### 故障检测

- `nPM1100 ERR` 引脚拉低 → 立即停止模拟
- LED 阵列闪烁 "ERR" 字样，LED_STATUS 快闪（100ms 周期）
- ERR 解除后恢复模拟
- ERR 具体原因见[nPM1100_PS_v1.5](/Docs/nPM1100_PS_v1.5.pdf)数据单

## 动画系统（class PixPillAnim）

| 动画 | 描述 | 原理 |
|------|------|------|
| **开机** | "PIXPILL" 滚动字幕 | 窗口滑动 42 帧位图 |
| **关机** | 全亮 → 逐行熄灭 | 从顶部往下一行一行熄灭 |
| **充电** | 电池框常亮 + 填充呼吸 | 1.5 秒周期，PWM 批量更新 |
| **故障** | "ERR" 全屏闪烁 | 250ms on/off，PWM 只设一次 |

- 动画通过**时间驱动**（HAL_GetTick）而非帧率驱动，不受主循环帧数影响
- LED 矩阵使用 18 位 uint32_t 位图描述（6 列 × 3 个 uint32_t）
- **查找表优化**：LED 编号映射表在初始化时生成一次，后续 O(1) 查找

## LED 开路/短路检测

IS31FL3736 内置每个 LED 的独立开路/短路检测功能，固件在启动时执行一次全 LED 检测。

### 检测流程

| 寄存器 | 地址 | 说明 |
|--------|------|------|
| Open Register | PG0 18h-2Fh | 24 字节，每偶数（0，2，4，6）比特对应一个 LED 开路状态 |
| Short Register | PG0 30h-47h | 24 字节，每偶数比特对应一个短路状态 |

寄存器位布局与 LED On/Off Register（00h-17h）完全一致：每个 SW 占 2 字节，D0/D2/D4/D6 对应 CS1-CS4（低地址）和 CS5-CS8（高地址）。

1. 设置 GCC = 0x01
2. 点亮所有 LED（关闭的 LED 不会刷新检测数据）
3. Configuration Register OSD 位 0 -> 1 (写0x04) 触发单次检测
4. 等待 ≥ 2 个扫描周期（5ms）
5. 读取 Open/Short 寄存器 → 解析为 96 个布尔值，真为错误
6. 恢复 LED 关闭状态和原始 GCC

### 结果处理

- 检测成功后（`valid == true`）→ 遍历 96 个 LED，统计开路/短路数量，1# 胶囊下跳过索引 0-5（物理上不存在）
- 存在故障 → 播放 ERR 动画 3 秒后继续正常运行
- 全部正常 → 静默通过，进入主循环

## 固件版本适配（条件编译）

PixPill 有两个胶囊版本（分别适配000#和1#胶囊），通过条件编译 PIXPILL_SIZE_1_CAPSULE 切换。

### 版本差异

| 参数 | 000# | 1# |
| ---- | ---- | ---- |
| 直径 | 6.6mm | 8mm |
| LED 数量 | 96 | 90 |
| 物理按键 | 背部，和其它元件位于同一面 | 顶部占用 row 0-1（6 颗 LED） |

### 切换方式

- 编辑 Components/Inc/PIXPILL_VERSION_SELECT.h，解注释 #define PIXPILL_SIZE_1_CAPSULE
- 或者直接在 Makefile 第 116 行解注释 -DPIXPILL_SIZE_1_CAPSULE

### 适配细节

原则：保留 96 索引不变，将不存在的 LED 视为"墙体"。

- `LIQUID_LED_MASK`：1# 下 row 0-1 全部置 false，液体粒子初始化和密度场计算自动跳过
- `LED_NEIGHBORS[96][8]`：1# 下索引 0-5 的 8 个方向邻居全部设为 -1（墙体），沙子无法移动到这些位置
- `SandSim::init()`：1# 下随机范围从 0-95 改为 6-95
- 动画/液体：无需改动，LIQUID_LED_MASK 自动过滤

其余模拟代码（扫描表、粒子数组、动画帧等）完全不变。

## 开发环境

### 推荐配置

本项目使用**VSCode + EIDE 插件**进行开发，这也是推荐的方式。

- **EIDE (Embedded IDE)**：VSCode 嵌入式开发插件，提供项目管理、编译、烧录一体化支持
  - 安装：VSCode 扩展市场搜索 "EIDE"
  - 导入项目：导入 PixPill 文件夹或打开PixPill.code-workspace工作区文件即可
- **STM32CubeMX**：生成 HAL 初始化代码和 `.ioc` 工程文件
- **arm-none-eabi-gcc**：交叉编译工具链
- **ST-LINK / J-Link / DAPLink**：调试烧录器

### 编译烧录

**方式一：EIDE（推荐）**

在 VSCode 中使用 EIDE 插件，点击 Build 按钮即可编译。

**方式二：GNU Make**

```bash
cd PixPill
make -j$(nproc)
```

连接调试器后，点击 Flash 按钮或使用 CubeProgrammer 烧录。

## 桌面模拟器

所有粒子算法可以先在 Python 桌面模拟器上验证和调参，避免反复烧录调试。

| 脚本 | 用途 |
|------|------|
| `sand_visual_sim.py` | 沙粒 8 方向模拟，邻居表 + 扫描表可视化 |
| `liquid_visual_sim.py` | 液体 SPH 参数调试，密度场/PWM 双窗口可视化 |

Python 模拟器输出与 C++ 固件的 LED 阵列一一对应，参数可在 Python 端调好后直接复制到 C++ 代码中。
